AI Bulk Validation - Rulebook

This document reflects the current business-rule behavior in src/validation/rules.js. Any non-exempt rule violation leads to Rejected status.

1. Exemption Behavior

Exemptions are active. If a rule key exists in exempted_fields[*].fields, that specific rejection reason is skipped.

  • Scope: Exemptions work per rule key (for example mrp, target_market, weights).
  • Effect: Validation still runs, but exempted rule errors are not appended to reasons.

2. Core Product Fields

These fields must be present and non-empty unless exempted:

  • GTIN Number
  • Brand Name
  • Product Name
  • Product Description
  • Category
  • Sub Category
  • Company Name
  • HS Code
  • IGST

3. Date Rules

Dates are considered valid only in YYYY-MM-DD format:

  • Activation Date (product level)
  • Deactivation Date (product level)
  • MRP Activation Date (first MRP row)

4. MRP and Market Logic

MRP checks are applied on the first MRP row: product.mrp[0].

  • Target Market: Mandatory.
  • MRP Location: Mandatory.
  • MRP Activation Date: Must be valid date.
  • Market-specific MRP value rule: If target_market = India, MRP value must be positive (> 0). If target market is non-India, MRP value is not mandatory.

5. Weights and Measures Logic

  • Unitized products bypass: If net content unit is each/piece/pieces/nos, all gross/net weight validations are skipped.
  • Otherwise mandatory: gross_weight, net_weight, gross_weight_unit, and net_weight_unit must all be provided.
  • Comparative rule: Gross must be greater than or equal to net when both sides can be converted and are in the same dimension.
  • Supported mass units: g, gram(s), kg, kilogram(s).
  • Supported volume units: ml, milliliter/millilitre, l, lt, liter/litre.
  • Non-comparable units: If unit conversion is not supported or dimensions differ, numeric gross-vs-net comparison is not applied.

6. Food-Specific Rules (Category = Food)

  • FSSAI Number: Required, length 14, must start with 1 or 2.
  • Food Type: Required.
  • Shelf Life: Value, unit, and based_on are all required.
Implementation Note
This rulebook is aligned with runtime behavior in src/validation/rules.js and should be revised whenever business-rule logic changes.
